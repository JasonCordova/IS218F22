AAA Testing

AAA unit testing, otherwise known as the Arrange-Act-Assert strategy, is a primarily efficient way to structure test cases in projects. It has become an essential standard in the industry of Computer Science, so learning it is just as essential in order to secure a job, but to also ensure that your code is fully functional and efficient.

In simpler terms:

1. The arrange section starts with having code that is required to set up a specific test. Usually, an instance of an object is created in this stage just for mock purposes.

2. The act portion requires a method from the object to be utilized, acting on some form of data that the method performs an action on.

3. The assert section is the simplest part, acting almost as a conditional statement. We are tasked with checking if the final results meet our expectations (in simpler terms, if the data is what we expect it to be after the method and if it is functional).
