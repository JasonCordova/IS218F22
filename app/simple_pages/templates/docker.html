Docker: Explanation


Containers

Docker specializes in allowing users to package their code and applications into standard units called containers. Containers are executable components that combine source code and any dependencies to produce a reliable and time-efficient application that can run from one environment to another. Through the use of containers, certain applications that may only work on one operating system may work on others without the use of a virtual machine as well.

Images

Containers run through a specific set of instructions otherwise known as images. Images are automatically generated when a Docker container is established, which produces an immutable (read-only) file of steps required to execute an application on Docker. One user can make an image for the purpose of one project and re-use it for another (and even on another machine), saving an immense amount of time on troubleshooting. Images are known as the “snapshot images” of a Docker container and contain all the dependencies required to run the application.

Docker Commands

  • 'docker --version' shows the current version of Docker
  • 'docker pull' pulls images from a Docker repository
  • 'docker run' creates a container from an image
  • 'docker exec' accesses the running container
  • 'docker ps' displays all running containers on Docker
  • 'docker ps -a' displays all active/running containers and exited ones
  • 'docker build' creates an image from
  • 'docker stop' stops running the container
  • 'docker kill' stops execution and kills the container
  • 'docker commit' creates new image of container
  • 'docker rm' deletes a stopped container
  • 'docker login' shows login prompt to Docker
  • 'docker images' shows all Docker images stored locally

Starting Project Using Terminal

1. Open your terminal (cmd, PowerShell, Linux terminal)
2. Using the 'cd' command, change the directory to where the project is stored or where you'd like to store it.
3. If no project exists at this time, use the 'git clone' command or 'docker pull' command to retrieve the project.
4. Use the 'docker build -t' function to build the project on Docker.
5. After building the project, you can set the port using 'docker run -dp 3000:3000'.
6. Once set, you can successfully use the 'docker -it sh' command to run commands in the container.

Accessing Container Using Terminal

1. Open your terminal (cmd, PowerShell, Linux terminal)
2. Using the 'docker ps -a' command, retrieve and copy the container id of the image you want to access.
3. Run 'docker exec -it bash', replacing container_id with the copied container id.
